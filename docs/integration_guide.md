# 연동 가이드

이 스킬팩은 세 가지 모드로 사용할 수 있다.

---

## Mode 1: 수동 검토

캠페인 정보를 직접 붙여넣고 검토를 요청하는 방식이다.
별도 설정 없이 바로 사용할 수 있다.

### 복붙용 프롬프트

```text
아래 캠페인을 no-more-tankday의 marketing-cultural-risk 스킬 기준으로 검토해줘.
가능하면 오늘 기준 실시간 이슈와 충돌 가능성도 확인해줘.

브랜드: [브랜드명]
캠페인명: [캠페인명]
제품명: [제품명]
출시일: [날짜]
메인 카피: [카피]
서브 카피: [서브 카피]
비주얼 설명: [비주얼 설명]
해시태그: [해시태그]
채널: [채널]
타깃 고객: [타깃]
```

## Mode 2: 최종 QA 게이트

마케팅 워크플로우에서 최종 출력 전에 자동으로 검수하는 방식이다.
에이전트의 시스템 프롬프트에 QA 프롬프트를 추가한다.

### 설정 방법

`agents/final_qa_prompt.md`의 내용을 에이전트의 최종 QA 단계 프롬프트에 붙여넣는다.

### 복붙용 프롬프트

```text
최종 결과물을 출력하기 전에 반드시 아래 기준으로 문화·역사·사회적 리스크를 검토해줘.
검토 기준: no-more-tankday의 marketing-cultural-risk 스킬

리스크 등급이 High 또는 Critical이면 최종 승인하지 말고, 문제 요소와 수정안을 제시해줘.
가능하면 실시간 이슈도 확인해줘.
```

---

## Mode 3: 생성 → 검수 → 수정 루프

에이전트가 콘텐츠를 만들고, 검수하고, 위험한 부분을 수정한 뒤 다시 검수하는 방식이다.

### 흐름

```
1. 카피/비주얼 생성
   ↓
2. marketing-cultural-risk 스킬로 검수
   ↓
3. 리스크 발견 시 수정
   ↓
4. 재검수
   ↓
5. Low/Medium 확인 후 최종 출력
```

### 복붙용 프롬프트

```text
캠페인 콘텐츠를 생성한 뒤, no-more-tankday의 marketing-cultural-risk 스킬 기준으로 자동 검수해줘.
리스크가 High 이상이면 수정하고 다시 검수하는 루프를 실행해줘.
최종 등급이 Low 또는 Medium일 때만 결과를 출력해줘.
```

---

## 도구 유무에 따른 차이

| 환경 | 검토 범위 |
|------|-----------|
| 검색 도구 없음 | 고정 리스크만 검토 (역사적 날짜, 민감 표현, 조합 리스크) |
| 검색 도구 있음 | 고정 리스크 + 실시간 이슈까지 검토 |

검색 도구가 없는 환경에서는 결과에 "실시간 이슈는 확인하지 못함"이라고 명시한다.
