# 연동 가이드

이 스킬팩은 마크다운 파일이라서 **텍스트를 읽을 수 있는 어떤 AI 에이전트에서든** 쓸 수 있다.

---

## 플랫폼별 설정 방법

### Claude Code

프로젝트 폴더의 `CLAUDE.md`에 아래 내용을 추가한다:

```md
## 마케팅 리스크 검수

캠페인 콘텐츠를 출력하기 전에 반드시 skills/marketing-cultural-risk/SKILL.md 기준으로 문화·역사·사회적 리스크를 검토한다.
리스크 등급이 High 이상이면 수정 후 재검수한다.
```

이후 대화에서 "이 캠페인 검수해줘"라고만 말하면 자동으로 검토한다.

### ChatGPT / GPTs

GPTs를 만들 때 `SKILL.md`와 `final_qa_prompt.md`를 **지식 파일(Knowledge)**로 업로드한다.

시스템 프롬프트에 추가:

```text
당신은 마케팅 리스크 검수 에이전트입니다.
캠페인 콘텐츠를 받으면 knowledge에 있는 SKILL.md 기준으로 문화·역사·사회적 리스크를 검토합니다.
```

### Claude Projects (웹앱)

claude.ai에서 프로젝트를 만들고, `SKILL.md`와 `final_qa_prompt.md`를 **프로젝트 지식(Project Knowledge)**에 업로드한다.

프로젝트 지침에 추가:

```text
이 프로젝트에서는 마케팅 콘텐츠 검수 시 skills/marketing-cultural-risk/SKILL.md 기준을 따른다.
```

### Cursor / Windsurf

프로젝트 폴더에 `.cursorrules` 또는 `.windsurfrules` 파일을 만들고, `SKILL.md` 내용을 넣는다.

```md
마케팅 콘텐츠 검수 시 skills/marketing-cultural-risk/SKILL.md 기준을 따른다.
```

### n8n / Make / 자동화 도구

AI 에이전트 노드의 시스템 프롬프트에 `final_qa_prompt.md` 내용을 넣는다.

콘텐츠 생성 노드 → 검수 노드 → 출력 노드 순서로 워크플로우를 연결하면 자동 검수가 된다.

---

## 3가지 사용 모드

### Mode 1: 수동 검토

캠페인 정보를 직접 붙여넣고 검토를 요청하는 방식이다.
어떤 플랫폼에서든 바로 쓸 수 있다.

```text
아래 캠페인을 no-more-tankday의 marketing-cultural-risk 스킬 기준으로 검토해줘.
가능하면 오늘 기준 실시간 이슈와 충돌 가능성도 확인해줘.

브랜드: [브랜드명]
캠페인명: [캠페인명]
제품명: [제품명]
출시일: [날짜]
메인 카피: [카피]
서브 카피: [서브 카피]
비주얼 설명: [비주얼 설명]
해시태그: [해시태그]
채널: [채널]
타깃 고객: [타깃]
```

### Mode 2: 최종 QA 게이트

마케팅 워크플로우에서 최종 출력 전에 자동으로 검수하는 방식이다.
에이전트의 시스템 프롬프트에 QA 프롬프트를 추가한다.

```text
최종 결과물을 출력하기 전에 반드시 아래 기준으로 문화·역사·사회적 리스크를 검토해줘.
검토 기준: no-more-tankday의 marketing-cultural-risk 스킬

리스크 등급이 High 또는 Critical이면 최종 승인하지 말고, 문제 요소와 수정안을 제시해줘.
가능하면 실시간 이슈도 확인해줘.
```

### Mode 3: 생성 → 검수 → 수정 루프

에이전트가 콘텐츠를 만들고, 검수하고, 위험한 부분을 수정한 뒤 다시 검수하는 방식이다.

```text
캠페인 콘텐츠를 생성한 뒤, no-more-tankday의 marketing-cultural-risk 스킬 기준으로 자동 검수해줘.
리스크가 High 이상이면 수정하고 다시 검수하는 루프를 실행해줘.
최종 등급이 Low 또는 Medium일 때만 결과를 출력해줘.
```

## 도구 유무에 따른 차이

| 환경 | 검토 범위 |
|------|-----------|
| 검색 도구 없음 | 고정 리스크만 검토 (역사적 날짜, 민감 표현, 조합 리스크) |
| 검색 도구 있음 | 고정 리스크 + 실시간 이슈까지 검토 |

검색 도구가 없는 환경에서는 결과에 "실시간 이슈는 확인하지 못함"이라고 명시한다.
