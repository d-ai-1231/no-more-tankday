# 출시 전 리스크 검토 워크플로우

캠페인 출시 전 문화·역사·사회적 리스크를 검토하는 표준 워크플로우다.

---

## 흐름

```
1. 캠페인/콘셉트 생성
   ↓
2. 카피/비주얼/콘티 생성
   ↓
3. marketing-cultural-risk 스킬로 고정 리스크 검수
   ↓
4. 가능한 경우 실시간 이슈 검토
   ↓
5. 리스크 등급 판정
   ↓
6. High/Critical이면 수정 루프
   ↓
7. 재검수
   ↓
8. Low/Medium 이하에서만 최종 승인
```

---

## 리스크 등급별 처리 방식

| 등급 | 처리 |
|------|------|
| **Low** | 그대로 진행 가능 |
| **Medium** | 수정 권장, 맥락 설명 필요 |
| **High** | 수정 전 최종 승인 금지 |
| **Critical** | 출시 중단, PR/법무/임원 검토 필요 |

---

## 복붙용 프롬프트

### 검토 요청

```text
아래 캠페인을 no-more-tankday의 marketing-cultural-risk 스킬 기준으로 검토해줘.

캠페인명: [캠페인명 입력]
제품명: [제품명 입력]
출시일: [날짜 입력]
메인 카피: [카피 입력]
서브 카피: [서브 카피 입력]
비주얼 설명: [비주얼 설명 입력]
해시태그: [해시태그 입력]
채널: [채널 입력]
타깃 고객: [타깃 입력]

가능하면 오늘 기준 실시간 이슈와 충돌 가능성도 확인해줘.
```

### 수정 후 재검토

```text
이전 검토에서 발견된 리스크를 수정했어. 다시 검토해줘.

수정된 캠페인명: [수정된 이름]
수정된 카피: [수정된 카피]
수정된 일정: [수정된 일정]
기존 리스크 등급: [이전 등급]
```
