# 실시간 이슈 검토 워크플로우

출시 직전 실시간 사회적 이슈와의 충돌 가능성을 검토하는 워크플로우다.

---

## 고정 리스크 vs 실시간 리스크

### 고정 리스크

스킬 파일 기준으로 언제든 검토할 수 있는 리스크:

- 역사적 기념일
- 추모일
- 참사일
- 민감 표현
- 조합 리스크

### 실시간 리스크

스킬 파일만으로는 알 수 없고, 검색 도구가 필요:

- 오늘/내일 발생한 사고나 참사
- 갑자기 논란이 된 표현
- SNS에서 확산 중인 밈이나 조롱
- 정치/사회적 갈등
- 지역 이슈
- 브랜드/인물 관련 실시간 논란

> **이 스킬팩 자체는 실시간 데이터를 직접 조회하지 않는다.**
> 웹 검색, 뉴스 검색, SNS 검색, 트렌드 검색 도구가 있는 에이전트 환경에서만 실시간 검토가 가능하다.

---

## 지원 검색 도구

| 도구 | 용도 | 호출 예시 |
|------|------|-----------|
| Tavily API | 웹+뉴스 통합 검색 | `tavily_search(query, search_depth="advanced", topic="news", days=3)` |
| Brave Search API | 웹+뉴스 검색 | `brave_web_search(query, freshness="pd", count=10)` |
| 웹 검색 (범용) | 일반 웹 검색 | `web_search(query + " site:news.naver.com OR site:news.daum.net")` |

## 검색 실행 흐름

### Step 1: 키워드 조합 생성

캠페인 정보를 기반으로 아래 6개 검색 쿼리를 만든다:

```
세트 1: "{브랜드명} 논란 OR 사고 OR 불매 OR 사과"
세트 2: "{캠페인명} 논란 OR 비판 OR 반응"
세트 3: "{제품명} 논란 OR 이슈"
세트 4: "{메인 카피 핵심 단어} 논란 OR 사고 OR 참사"
세트 5: "{출시일} 사고 OR 참사 OR 사건"
세트 6: "{출시 지역} 사고 OR 참사 OR 이슈"
```

### Step 2: 검색 실행

각 쿼리를 순서대로 실행한다. 병렬 실행이 가능하면 동시에 실행한다.

### Step 3: 결과 필터링

- **최근 3일 이내** 기사만 유효하다.
- 3일 이상 된 기사는 무시한다.
- 네이버 뉴스, 다음 뉴스, 주요 일간지, SNS 게시글을 우선 확인한다.

### Step 4: 충돌 분석

검색 결과에서 아래 항목이 발견되면 리스크 등급을 상향한다:

| 발견 시 | 상향 기준 |
|---------|-----------|
| 브랜드/캠페인명 관련 부정 기사 | Medium → High |
| 출시일에 대형 사고/참사 발생 | High → Critical |
| 유사 표현으로 논란이 된 사례 | Medium 이상 |
| 불매 운동 또는 대규모 비판 | Critical |
| 정치적 갈등과 직접 연관 | High 이상 |

### Step 5: 결과 출력

아래 형식으로 출력한다.

## 출력 형식

```md
# 실시간 이슈 검토 결과

## 검색 가능 여부
가능 / 불가능

## 사용한 검색 도구
Tavily / Brave Search / 웹 검색 / 없음

## 실행한 검색 쿼리
1. "{브랜드명} 논란 OR 사고 OR 불매 OR 사과"
2. "{캠페인명} 논란 OR 비판 OR 반응"
3. ...

## 확인한 결과 (최근 3일 이내)
-
-

## 발견된 실시간 이슈
-
-

## 캠페인과의 충돌 가능성
Low / Medium / High / Critical

## 리스크 상향 여부
상향 없음 / Medium으로 상향 / High로 상향 / Critical로 상향

## 권고
진행 / 수정 / 보류 / 중단
```

## 검색 도구가 없는 경우

결과에 아래 문장을 명시한다:

> "실시간 이슈 검토는 수행하지 못했으며, 고정 리스크 기준으로만 판단했습니다."

이 경우 캠페인 담당자가 직접 뉴스·SNS를 확인한 후 출시를 진행해야 한다.
