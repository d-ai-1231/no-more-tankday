# marketing-cultural-risk

## 목적

이 스킬은 마케팅 캠페인 공개 전에 캠페인명, 제품명, 날짜, 카피, 비주얼, 해시태그, 콘티 흐름이 한국 사회의 역사·문화·정치·사회적 맥락과 충돌할 가능성을 검토한다.

핵심 목적은 브랜드 의도를 판단하는 것이 아니라, 대중·언론·피해자 커뮤니티·시민단체·임직원이 어떻게 받아들일 수 있는지 사전에 점검하는 것이다.

---

## 사용 시점

다음 경우 반드시 사용한다:

- 캠페인명 확정 전
- 카피 초안 생성 후
- 이벤트 날짜 확정 전
- SNS/앱푸시/랜딩페이지 공개 전
- 콘티/스토리보드 최종안 제출 전
- 한국 시장용 로컬 캠페인 공개 전
- 최종 QA 단계

| 캠페인명 | 필수 |
| 제품명 | 권장 |

| 프로모션 기간 | 권장 |

## 검토 기준

### 1. 날짜 리스크

출시일, 이벤트 날짜, 프로모션 기간이 한국의 역사적·정치적·추모·참사·사회적 민감일과 겹치는지 확인한다.

### 2. 네이밍 리스크

캠페인명이나 제품명이 다음을 연상시키는지 확인한다:

- 폭력
- 군사 행동
- 국가폭력
- 참사
- 죽음
- 차별
- 조롱
- 역사적 상처
- 정치적 구호
- 피해자 또는 가해자 관련 표현

### 3. 카피 연상 리스크

카피가 다음과 닮았거나 연상되는지 확인한다:

- 역사적 사건의 유명 문구
- 정치 구호
- 시위 구호
- 참사 관련 표현
- 피해자 관련 표현
- 고문, 진압, 폭력, 차별, 조롱을 떠올리게 하는 표현

### 4. 비주얼 리스크

비주얼이 날짜, 캠페인명, 카피와 결합했을 때 부정적 의미를 강화하는지 확인한다.

### 5. 조합 리스크

개별 요소만 보지 말고 반드시 조합으로 검토한다.

검토해야 할 조합:

- 날짜 + 캠페인명
- 날짜 + 제품명
- 날짜 + 카피
- 제품명 + 카피
- 비주얼 + 카피
- 해시태그 + 사회적 사건
- 콘티 흐름 + 사회적 맥락

> 리스크는 단어 하나보다 조합에서 자주 발생한다.

### 6. 대중 반응 시뮬레이션

다음 관점에서 어떻게 비판받을 수 있는지 시뮬레이션한다:

- 일반 소비자
- SNS 사용자
- 언론
- 피해자 단체
- 시민단체
- 정치권
- 임직원
- 브랜드 파트너

## 실시간 이슈 확인

이 스킬 자체는 실시간 뉴스나 SNS 데이터를 직접 조회하지 않는다.

사용 중인 에이전트가 아래 검색 도구에 접근할 수 있는 경우, 최종 승인 전 반드시 실시간 이슈 검토를 수행한다.

### 지원 검색 도구

| 도구 | 용도 | 호출 방식 |
|------|------|-----------|
| Tavily API | 웹+뉴스 통합 검색 | `tavily_search(query, search_depth="advanced", topic="news", days=3)` |
| Brave Search API | 웹+뉴스 검색 | `brave_web_search(query, freshness="pd", count=10)` |
| 웹 검색 (범용) | 일반 웹 검색 | `web_search(query + " site:news.naver.com OR site:news.daum.net")` |

### 검색 실행 규칙

**Step 1: 핵심 키워드 조합 생성**

아래 6개 키워드 세트를 만들고, 각각 검색한다:

```
세트 1: "{브랜드명} 논란 OR 사고 OR 불매 OR 사과"
세트 2: "{캠페인명} 논란 OR 비판 OR 반응"
세트 3: "{제품명} 논란 OR 이슈"
세트 4: "{메인 카피 핵심 단어} 논란 OR 사고 OR 참사"
세트 5: "{출시일} 사고 OR 참사 OR 사건" (출시일이 포함된 최근 뉴스)
세트 6: "{출시 지역} 사고 OR 참사 OR 이슈"
```

**Step 2: 날짜 필터**

- 검색 결과는 반드시 **최근 3일 이내** 기사만 유효하다.
- 3일 이상 된 기사는 무시한다.

**Step 3: 결과 분석**

검색 결과에서 아래 항목이 발견되면 리스크 등급을 상향한다:

| 발견 시 | 상향 기준 |
|---------|-----------|
| 브랜드/캠페인명 관련 부정 기사 | Medium → High |
| 출시일에 대형 사고/참사 발생 | High → Critical |
| 유사 표현으로 논란이 된 사례 | Medium 이상 |
| 불매 운동 또는 대규모 비판 | Critical |
| 정치적 갈등과 직접 연관 | High 이상 |

**Step 4: 검색 도구가 없는 경우**

검색 도구에 접근할 수 없으면, 결과에 아래 문장을 명시한다:

> "실시간 이슈는 확인하지 못했으며, 고정 리스크 기준으로만 판단했습니다."

---

## 리스크 등급

| 등급 | 의미 | 처리 방식 |
|------|------|-----------|
| **Low** | 의미 있는 리스크 없음 | 진행 가능 |
| **Medium** | 오해 가능성 있음 | 수정 권장, 맥락 설명 필요 |
| **High** | 공개 비판 가능성 높음 | 수정 전 출시 금지 |
| **Critical** | 불매·언론 확산·피해자 커뮤니티 반발 가능성 | 출시 중단, PR/법무/임원 검토 필요 |

---

## Escalation Rule

- 날짜 리스크와 네이밍/카피 리스크가 함께 있으면 최소 High로 상향한다.
- 역사적 상처를 희화화하거나 소비하는 것으로 해석될 수 있으면 Critical로 상향한다.
- 실시간 사회적 이슈와 캠페인 표현이 충돌하면 최소 Medium 이상으로 상향한다.
- 실시간 참사/사고/정치적 갈등과 직접 충돌하면 High 또는 Critical로 상향한다.
