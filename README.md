# no-more-tankday

### 터지기 전에 막는다

마케팅 캠페인은 의도대로 만들어진다.
해석은 대중의 몫이다.

이 검수 가이드은 출시 전에 터질 조합을 찾아준다.

---

## 이 검수 가이드은 뭔가요?

콘텐츠를 만들어주는 도구가 아니다.
만들어진 콘텐츠가 한국 사회의 역사·문화·사회적 맥락과 부딪히는지 확인하는 **출시 전 검수 게이트**다.

캠페인명, 제품명, 출시일, 카피, 비주얼, 해시태그, 콘티 흐름을 통째로 본다.
단어 하나가 아니라 **조합**이 위험한지 판단한다.

---

## 왜 필요한가요?

"괜찮을 거야"라고 생각한 캠페인이, 특정 날짜에 특정 표현이 만나면서 전혀 다르게 읽힐 수 있다.

예를 들어볼게요.

| 날짜 | 캠페인명 | 카피 | 결과 |
|------|----------|------|------|
| 5월 18일 | Tank Day | 책상에 탁! | 즉시 중단 |
| 평일 | Summer Tumbler Week | 여름을 더 시원하게 | 문제 없음 |

같은 제품이라도 날짜와 표현이 바뀌면 의미가 완전히 달라진다.

---

## 뭘 검토하나요?

| 보는 것 | 왜 보는가 |
|---------|----------|
| 캠페인명 | 이름 자체가 논란이 될 수 있다 |
| 제품명 | 특정 사건·집단을 연상시킬 수 있다 |
| 출시일 | 역사적 민감일과 겹칠 수 있다 |
| 카피 | 유명 구호·참사 표현과 닮을 수 있다 |
| 비주얼 | 날짜·카피와 합쳐지면 의미가 바뀐다 |
| 해시태그 | SNS에서 확산 속도가 다르다 |
| 콘티 흐름 | 전체 맥락에서 부적절한 서사가 보일 수 있다 |

---

## 고정 리스크 vs 실시간 리스크

### 고정 리스크 (언제든 검토 가능)

역사적 기념일, 추모일, 참사일, 폭력·군사·국가폭력·차별·조롱 연상 표현, 날짜+카피 조합.

### 실시간 리스크 (검색 도구 필요)

오늘 아침에 터진 사고, 갑자기 논란이 된 단어, SNS에서 확산 중인 비판 여론.

> 이 검수 가이드은 실시간 뉴스를 직접 조회하지 않는다.
> 검색 도구가 연결된 에이전트 환경에서만 실시간 검토가 가능하다.

| 검색 도구 | 특징 |
|-----------|------|
| Tavily API | 웹+뉴스 통합 검색, 추천 |
| Brave Search API | 월 2,000회 무료 |
| 웹 검색 (범용) | Naver/Daum 뉴스 직접 검색 |

## 설치 (30초)

```bash
git clone https://github.com/d-ai-1231/no-more-tankday.git
cp -r no-more-tankday/skills/marketing-cultural-risk ./skills/
```

## 설정 방법

### 방법 A: 자동 검수 (추천)

`CLAUDE.md`에 한 줄 추가하면, 캠페인 콘텐츠를 만들 때마다 알아서 검수한다.

```bash
# setup/CLAUDE_SNIPPET.md 내용을 프로젝트 CLAUDE.md에 붙여넣기
```

이후 "여름 캠페인 만들어줘"라고 하면 에이전트가 카피를 만들고, 리스크까지 검수해서 결과를 같이 출력한다.

### 방법 B: 수동 검수 (슬래시 커맨드)

```bash
mkdir -p .claude/commands
cp setup/review-campaign-command.md .claude/commands/review-campaign.md
```

이후 `/review-campaign 브랜드명`으로 언제든 검수를 호출할 수 있다.

### 방법 C: 복붙

별도 설정 없이 `agents/final_qa_prompt.md` 내용을 직접 붙여넣어도 된다.

자세한 사용 모드(수동 검토 / QA 게이트 / 생성→검수→수정 루프)는 [integration_guide.md](docs/integration_guide.md)에서 확인.
